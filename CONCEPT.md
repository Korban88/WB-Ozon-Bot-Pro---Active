# WB/Ozon Card Bot — Концепция и архитектура (v2)

> Читай этот файл в начале нового диалога, чтобы сразу войти в контекст.

---

## Что за продукт и зачем он нужен

Telegram-бот для продавцов на Wildberries и Ozon. Продавец загружает фото товара — бот генерирует три вещи:

1. **Текст для карточки** — SEO-заголовок, описание, характеристики, хэштеги. Вставил в редактор маркетплейса и готово.
2. **5 Premium Visuals** — атмосферные сцены с товаром, без текста на изображении. Готовы к загрузке в слот доп.фото WB/Ozon или в рекламу.
3. **Ad Copy Pack** — рекламные хуки, тексты объявлений, бриф для UGC-видео. Для VK, Telegram, Яндекс или для блогера.

---

## Главное правило — товар не трогать

Это не просто ограничение, это архитектурный принцип.

Фото товара, которое загрузил пользователь — неприкосновенный объект. Пиксели, цвет, форма, логотип, принт, надписи на упаковке — всё остаётся ровно как на фото.

Что разрешено делать с фото товара:
- Отправить в OpenRouter для анализа и генерации текста (только чтение)
- Вырезать фон (только alpha-канал, пиксели товара не меняются)
- Наложить как слой поверх сгенерированной сцены

Что запрещено:
- Перегенерировать товар
- Отправлять фото товара в API генерации изображений (только OpenRouter)
- Изменять форму, цвет, принт, логотип

---

## Почему нет текста на изображениях

Это намеренное решение, принятое в v2.

Раньше бот пытался наносить заголовок и буллеты поверх изображения через Pillow. Это не работало:
- Pillow-типографика выглядит как автоматический шаблон
- Кернинг, иерархия, читаемость — всё слабое
- AI-модели тоже плохо рисуют текст внутри изображений

Решение: текст и изображение разделены полностью.
- Изображения — чистые, атмосферные, профессиональные
- Текст доставляется отдельно как форматированные сообщения

## Как работает диалог (шаги)

```
/start
  Пользователь выбирает маркетплейс (WB / Ozon)
  Пользователь выбирает категорию товара
  Пользователь вводит название товара
  Пользователь описывает преимущества товара
  Пользователь загружает фото товара
    → AI читает фото и текст → генерирует JSON-карточку
  Бот присылает текст карточки (Layer 1)
  Бот показывает три кнопки:
    [🖼 5 Premium Visuals]  → генерирует 5 сцен с товаром (Layer 2)
    [📣 Ad Copy Pack]       → генерирует рекламные тексты (Layer 3)
    [🔄 Новый товар]        → начать заново
```

---

## Как генерируются Premium Visuals

Для каждого из 5 типов карточек происходит следующее:

**Шаг 1.** Из фото товара извлекают доминирующие цвета (Pillow). Получают строку вроде `"warm terracotta and cream"` — это `color_mood`.

**Шаг 2.** Отправляют запрос в OpenAI gpt-image-1 с промтом: "сгенерируй пустую сцену-фон в таком-то стиле, в такой палитре". Товар в промт не попадает никогда. Получают фоновую картинку без товара.

**Шаг 3.** Из фото товара вырезают фон — остаётся RGBA-вырезка. Если rembg не установлен (а на VPS он отключён из-за памяти) — используется Pillow corner-sampling: берут среднее из 8 угловых точек, делают прозрачными похожие пиксели. Работает нормально для студийных фото.

**Шаг 4.** Накладывают RGBA-вырезку товара поверх сцены (Pillow). Добавляют мягкую тень от alpha-канала. Никакого текста. Сохраняют как PNG 800×1000.

Если OpenAI API key не указан — вместо сцены генерируется Pillow-градиент из цветов товара. Работает как fallback.

---

## Пять типов карточек

| Тип | Сцена | Где товар |
|---|---|---|
| hero | Студийный градиентный фон | Немного выше центра |
| lifestyle | Интерьер, натуральный свет | По центру |
| social | Улица / сад, soft bokeh | Немного ниже центра |
| editorial | Драматичный, кинематографичный | Выше центра |
| detail | Макро-фактура материала | По центру, крупно |

Все типы: 800×1000px, сцена заполняет весь кадр, текст на изображении отсутствует.

Набор типов зависит от категории товара. Например, для одежды: hero → lifestyle → social → editorial → detail. Для электроники: hero → editorial → lifestyle → social → detail.

## Как генерируется Ad Copy Pack

Один запрос в OpenRouter (текстовая модель, без изображения). Промт содержит: название товара, маркетплейс, категорию, описание преимуществ от продавца.

В ответ приходит JSON:
- `hooks` — 5 рекламных хуков (разные углы: проблема/решение, любопытство, результат, цифра, эмоция)
- `copy_short` — 3 коротких текста для объявлений
- `copy_medium` — 2 средних текста для постов
- `ugc_brief` — сценарий для 15–30 секундного видео с тайм-кодами

## Структура файлов

```
bot.py              Точка входа. Запускает polling, регистрирует роутеры, вызывает ensure_fonts().
config.py           Читает .env, валидирует обязательные переменные.
states.py           FSM-состояния диалога.
keyboards.py        Inline-клавиатуры для каждого экрана.

handlers/
  start.py          Обрабатывает /start. Сбрасывает состояние, показывает приветствие.
  dialog.py         Шаги 1–6: маркетплейс, категория, название, преимущества, фото, текст карточки.
  design.py         Шаги 7–8: генерация Premium Visuals и Ad Copy Pack.

services/
  openrouter.py     Два метода: generate_card() — текстовая карточка с vision.
                    generate_ad_copy() — рекламный копипак.
  scene_gen.py      Генерирует пустую фоновую сцену через gpt-image-1 /generations.
                    Товар сюда НЕ попадает.
  card_renderer.py  render_card(): сцена + товар → PNG. Без текста.
  card_types.py     Определяет 5 типов карточек: промты сцен, позиции товара, маппинг по категориям.
  color_extractor.py Извлекает доминирующие цвета из фото товара (Pillow K-means).
  product_cutout.py Удаляет фон: rembg (если есть) или Pillow corner-sampling.
  fonts.py          ensure_fonts() скачивает Montserrat при первом запуске. Шрифты не используются
                    в v2 для overlay, но остаются для возможного будущего использования.
  card_layout.py    Старые layout-классы. Не используются в основном pipeline. Можно удалить.

utils/images.py     send_step_image() — отправляет PNG-иллюстрацию шага диалога.
                    Если файл не найден — отправляет plain text (не ломает диалог).

assets/images/      PNG-иллюстрации для шагов диалога. Актуальные файлы:
                    welcome, choose_marketplace, choose_category, enter_title,
                    enter_benefits, upload_photo, generating, card_ready, error.

                    Мёртвые файлы (в коде больше не используются, можно удалить):
                    design_concepts.png, visual_concepts.png

assets/fonts/       Montserrat TTF. Скачиваются автоматически при первом запуске.
```

## Переменные окружения

| Переменная | Обязательна | Зачем |
|---|---|---|
| TELEGRAM_BOT_TOKEN | Да | Токен бота от @BotFather |
| OPENROUTER_API_KEY | Да | Текстовая генерация (карточка + копипак) |
| OPENAI_API_KEY | Нет | Генерация сцен через gpt-image-1. Без него — Pillow-градиент |
| OPENROUTER_MODEL | Нет | Модель для текста. Default: openai/gpt-4o-mini |
| LOG_LEVEL | Нет | Default: INFO |

## Что осталось от старой версии (v1)

**Убрано в v2:**
- Шаг "Сгенерировать 5 дизайн-концептов" — текстовые ТЗ с hex-цветами и типографикой. Был промежуточным шагом между текстовой карточкой и визуалами. Убран как лишний.
- Тип карточки `features` — чисто Pillow-инфографик с нарисованными иконками. Заменён на `social`.
- Pillow-типографика поверх всех визуальных карточек — заголовки, буллеты, шрифты на изображении.
- Кнопка "Изменить описание" после визуальных карточек.
- `after_design_keyboard` — кнопки после текстовых ТЗ.
- `design_concepts` state в FSM.

**Добавлено в v2:**
- Тип карточки `social` (outdoor / social media сцена).
- Full-height canvas: сцена заполняет весь 800×1000, нет текстовой панели снизу.
- `generate_ad_copy()` в openrouter.py.
- Хендлеры `cb_premium_visuals` и `cb_ad_copy` в design.py.
- Новые кнопки после карточки: Premium Visuals + Ad Copy Pack + Новый товар.

**Осталось из v1:**
- Весь диалог шагов 1–6 не изменился.
- Вся логика color_extractor, product_cutout, scene_gen — без изменений.
- card_renderer работает так же, просто без финального вызова типографики.
- generate_card() не изменился.

## Известные особенности

| Ситуация | Как работает |
|---|---|
| OPENAI_API_KEY не указан | Premium Visuals работают через Pillow-градиент вместо AI-сцен |
| rembg не установлен | Фон вырезается через Pillow corner-sampling (достаточно для студийных фото) |
| Файл изображения шага не найден | Бот отправляет plain text вместо фото — диалог не ломается |
| Длинная карточка > 4096 символов | Автоматически разбивается на несколько сообщений |
| JSON от AI с markdown-обёрткой | _parse_json_response() автоматически срезает ```json ... ``` |
